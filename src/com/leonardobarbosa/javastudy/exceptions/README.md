# Exceptions

1. flow → Demonstrates how exceptions propagate through the 
call stack and affect the execution flow of the application.

---

2. unchecked (RuntimeException) → Represents exceptions that 
occur at runtime and are not required to be explicitly handled.

---

3. checked →  Represents exceptions that must be handled 
or declared using `throws`.

---

4. throwing → Covers the use of `throw` and `throws` 
to explicitly create and propagate exceptions.

---

5. custom → Demonstrates how to create and use custom 
exceptions for specific application rules.

---

6. error → Represents serious problems that are not 
meant to be handled by applications.
